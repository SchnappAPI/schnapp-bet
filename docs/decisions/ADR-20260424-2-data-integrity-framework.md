# ADR-20260424-2: Data integrity and completeness framework

Date: 2026-04-24

## Context

The project has accumulated data-integrity gaps that were accepted as tolerable but never systematically tracked or resolved:

- 135 NBA players in `odds.player_map` are unmapped. Previously accepted as "historically inactive, not blocking grading." Not surfaced, not tracked, no path to resolution.
- `mlb.players` is truncate-reload scoped to the current season. 20–32% of historical `batter_id`/`pitcher_id` joins on `mlb.player_at_bats` and `mlb.career_batter_vs_pitcher` resolve to NULL names. Accepted via "join at read time."
- Sports ETL encounters upstream publication lag routinely. Nothing in the current system distinguishes "truly missing" from "hasn't appeared yet."
- No structured place where data-integrity issues surface. Problems are discovered ad-hoc.

Root causes of unexpected NULLs and the right response per cause:

1. Upstream publication lag — daily retry.
2. Transient API error — already handled by `get_engine` retry (3 attempts × 45s).
3. Entity mapping gap — mapping resolver (not retry; retrying a failed lookup is pointless).
4. Schema drift / new entity — daily retry (resolves after next reference-table pull).
5. Legitimately null (domain-conditional, e.g., no batted ball on a strikeout) — never retry; accept.
6. Our parsing bug — human attention; retry masks it.

A single generic "retry everything NULL" is wrong for multiple of these.

## Decision

Three-layer framework, each addressing a distinct failure mode.

**Layer 1 — Invariants enforced at write time.** Every table that opts in declares its must-not-be-null critical fields in `CRITICAL_FIELDS` (a Python dict in `shared/integrity.py`). Before each MERGE/INSERT, `validate_row()` checks each row. Violating rows go to `common.ingest_quarantine` (full row payload as JSON, failed invariant, source workflow, timestamps) — they do NOT land in the production table. ETL continues per-row (fail-soft).

Two kinds of "null is valid" are distinguished:
- **Stat-zero vs stat-null.** A stat column (PTS, REB, hit counts) is never NULL when the row exists. PTS=0 with MIN>0 is a player who did not score; PTS=NULL is a parsing failure. DNP players produce no row at all.
- **Domain-conditional nullability** declared as SQL-expression rules. Real examples: `nba.daily_grades.over_price` NULL on bracket lines (only center carries posted price); `opportunity_volume_grade`/`opportunity_expected_grade` NULL for non-threes markets; `mlb.play_by_play.hit_launch_speed/angle/distance` NULL when `is_hit_into_play = 0`; `mlb.play_by_play.batter_id` NULL on pickoff/caught-stealing events.

Any successful Layer 1 validation for `(table_name, row_key)` also clears prior `common.ingest_quarantine` and `common.data_completeness_log` entries for that same key, regardless of resolution path.

**Layer 2 — Mapping resolver.** Entity-matching gaps go to `common.unmapped_entities` keyed by `(source_feed, entity_type, source_key)`. Nightly `resolve-mappings.yml` attempts auto-resolution (exact match case-insensitive, last-name + first-initial, normalized string distance). Auto-resolutions logged. After 3 nights of attempts, raises a GitHub Issue labeled `data-integrity:unmapped`.

**Layer 3 — Retry cadence for upstream lag.** `common.data_completeness_log` keyed by `(table_name, row_key, column_name)` with `first_detected_at`, `last_attempt_at`, `attempt_count`, `resolved_at`. Nightly `retry-incomplete.yml` scans quarantine + production for declared-critical fields still NULL. Reattempts if `attempt_count < 3` AND `last_attempt_at < today` — re-fetches that specific row key, not a full ETL re-run. After 3 attempts, raises a GitHub Issue labeled `data-integrity:incomplete`.

**Retry cadence: one per day, max 3 attempts.** Sports data publication cycles are daily, not sub-daily. 3-day cap survives weekend publication delays. Beyond 3 days, the root cause is almost never upstream lag.

**Surfacing: `docs/HEALTH.md` + auto-opened Issues.** HEALTH.md regenerated by `daily-health-report.yml` (in sports-modeling) or on-demand via `/skill regenerate-health` (in schnapp-bet, per `docs/decisions/ADR-20260517-1-bootstrap-strategy.md` lineage and D5 of the scaffolding ADR). Contains:
- Column-level integrity from Layer 1: quarantine counts per table, unresolved mappings, incomplete-retry counts.
- Relational integrity from scanned queries (`RELATIONAL_CHECKS` sibling dict in `shared/integrity.py`). Examples: NBA schedule with status≥1 should have `nba.daily_lineups` rows covering both teams' rosters; MLB games with status='F' should have corresponding `batting_stats` and `pitching_stats`; NBA schedule with status=3 should have box-score rows for every player flagged non-Inactive.

**Downstream-inference safety rule (documented so sessions do not re-derive it wrong):** "did this player play?" is answered by `minutes > 0` in `nba.player_box_score_stats` or by `starter_status != 'Inactive'` in `nba.daily_lineups`. NEVER by whether stat values are zero. PTS=0 with MIN>0 is a player who played and did not score; PTS=0 with MIN=0 is a player who was on the roster but did not take the floor. Any code using stat-zero as a non-participation proxy is wrong.

Issues auto-close via the Layer-1 clearing rule. `permissions: issues: write` on the workflow YAML scopes the default `GITHUB_TOKEN`.

## Consequences

- New tables: `common.ingest_quarantine`, `common.unmapped_entities`, `common.data_completeness_log`. DDL owned by `shared/integrity.py`.
- New workflows: `resolve-mappings.yml`, `retry-incomplete.yml`, `daily-health-report.yml` (the last is gitignore-only in schnapp-bet per D5).
- New shared module: `shared/integrity.py` with `CRITICAL_FIELDS` catalog + `validate_row()` + `write_quarantine()` + `move_from_quarantine()`.
- Every existing ETL script gets a one-line change to route rows through `validate_row()` before upsert. Per-sport rollout.
- Every `common.*`, `nba.*`, `mlb.*`, `nfl.*` table must declare its critical fields in `CRITICAL_FIELDS`. 1–2 hours per sport done carefully (including conditional-nullability rules).
- `docs/HEALTH.md` becomes a canonical artifact (sports-modeling pattern). In schnapp-bet, gitignored; on-demand via skill.
- 135 NBA unmapped players become a resolvable class or get marked `legitimate_unresolvable = 1`.
- `mlb.players` 20–32% NULL rate is a read-time join problem, addressed by ADR-20260501-3 Initiative D.

**First-run scope requirement:** v1 must include a retroactive scan that applies each table's `CRITICAL_FIELDS` rules against existing production rows, writes violations to `common.data_completeness_log` with `detected_retroactively = 1`, but does NOT move existing rows out of production. Forward-looking quarantine applies only to new writes.
