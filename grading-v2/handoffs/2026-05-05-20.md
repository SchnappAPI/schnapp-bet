## What was done

Phase 7 adds `common.model_performance` to `grading/weekly_calibration.py`. Three new functions:

- **`ensure_model_performance_table(engine)`** ‚Äî DDL guard, creates table on first calibration run if absent. Schema: `perf_id` (IDENTITY PK), `snapshot_date`, `model_version`, `market_group`, `n_resolved`, `hit_rate`, `avg_ev_pct`, `profit_weighted_score`, `composite_grade_correlation`, `created_at`. UNIQUE constraint on `(snapshot_date, market_group)`.

- **`fetch_resolved_for_performance(engine, window_days)`** ‚Äî queries `common.daily_grades` for resolved Over rows in the calibration window. Fetches `composite_grade` (pre-v2 baseline), `model_prob`/`implied_prob`/`ev_pct` (Phase 5 columns, NULL for older rows), and `outcome_binary`.

- **`write_model_performance(engine, df, model_version)`** ‚Äî computes per-market-group metrics and upserts. Called from `main()` after `fit_logistic_models`. Three rows per run (Volume / Rate / Counting). Idempotent DELETE WHERE snapshot_date + INSERT.

**Metrics computed:**
- `n_resolved` ‚Äî row count per group
- `hit_rate` ‚Äî fraction of Over outcomes that Won
- `avg_ev_pct` ‚Äî mean of stored ev_pct (NULL if all NULL)
- `profit_weighted_score` ‚Äî sum(ev_pct) where model_prob > implied_prob (0.0 when no active weights)
- `composite_grade_correlation` ‚Äî Pearson correlation between composite_grade and outcome_binary (NULL if < 2 valid rows)

---

## Key invariants

- UNIQUE on `(snapshot_date, market_group)` ‚Äî re-running calibration on the same day overwrites via DELETE + INSERT.
- `ev_pct`, `model_prob`, `implied_prob` are NULL for rows graded before Phase 5. Metrics degrade gracefully: `avg_ev_pct=NULL`, `pws=0.0`, `cgc=NULL`.
- 0 rows expected in `profit_weighted_score` until logistic `grade_weights` corpus accumulates (~2 weeks of resolved Phase 2 grades).
- `composite_grade_correlation` serves as the historical baseline comparison metric for Phase 8 display.
